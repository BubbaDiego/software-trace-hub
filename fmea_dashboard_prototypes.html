OS Kernel — OSE (Modules: LVP, SYR, PCA, ETCO2)
OSE Scheduling
Task starvation — a lower-priority task never executes because higher-priority tasks monopolize CPU
Mitigation: Software watchdog monitors critical tasks; hardware watchdog resets module on timeout. Bounded execution + priority ceiling protocol prevents priority inversion.
OSE Memory Management
Memory leak or corruption — dynamic allocation fails or overwrites adjacent memory
Mitigation: Static memory allocation at init; no runtime malloc. Memory protection units isolate task address spaces. CRC on critical data structures.
OSE Interrupt Management
Interrupt storm or lost interrupt — hardware interrupt handler takes too long or drops events
Mitigation: Interrupt handlers are minimal (deferred processing). Watchdog detects stalled interrupt context. Nested interrupt priority scheme with bounded latency.
OS Kernel — SMX (PCU)
SMX Scheduling
Priority inversion or task deadlock in PCU real-time scheduler
Mitigation: Priority inheritance protocol; deadlock detection via watchdog. Critical sections bounded to < 100μs.
SMX Messaging
Message queue overflow or corruption in inter-task communication
Mitigation: Bounded message queues with overflow detection. CRC on message payloads. Fallback to safe state on queue corruption.
SMX System Clock
Timer drift or failure — system tick counter wraps or stops
Mitigation: Dual clock sources with cross-check. Hardware watchdog independent of system clock. Timer wrap-safe arithmetic in all time calculations.
